# Diceverse v0.0.1a Roadmap

## Goal

Ship a simple local-first React SPA that proves the Diceverse MVP loop:

1. roll dice
2. resolve payouts and tricks
3. earn money, XP, and turn points
4. buy and upgrade dice between turns
5. survive by generating extra turns

## Release Scope

`v0.0.1a` should be playable locally in the browser with one persistent save slot and a complete end-to-end run loop.

Not in scope for this cut:

- online accounts
- cloud saves
- audio
- animation polish
- balance depth beyond basic pacing sanity
- advanced meta systems beyond the first permanent progression hooks

### 2. Core Game Types

Status: Completed on 2026-04-21.

- define TypeScript models for dice, run state, trick definitions, rewards, and meta progression
- separate persistent save data from run-local state
- version the save shape from the start

Implementation details:

- added `src/game/types.ts` for the core domain model: dice, mods, tricks, meta progress, run state, and save metadata
- added `src/game/content.ts` for the MVP trick catalog and mod pool definitions
- added `src/game/bootstrap.ts` for save versioning and typed starter-state factories
- split persistent account progress from run-local state in the `DiceverseSave` shape
- switched the shell to render from the typed bootstrap state instead of inline mock literals

Exit criteria:

- all gameplay state has explicit types
- save data can evolve without rewriting the whole app

### 3. Turn Resolution Engine

Status: Completed on 2026-04-21.

- implement spend-turn, roll, base payouts, trick detection, multiplier handling, XP gains, and turn-point conversion
- keep the resolver deterministic from a provided RNG source so it can be tested

Implementation details:

- implemented spend-turn, die rolling, persistent die-mult updates, base payouts, trick detection, extra rewards, XP gain, and turn-point conversion
- applied immediate meta updates after each turn by recalculating player level and unlocked tricks
- stored a structured last-turn summary and appended readable log entries for the UI
- wired the roll button in `src/App.tsx` to real resolver state instead of placeholder shell behavior

Exit criteria:

- one button can fully resolve a turn
- results match the MVP design rules for base income and unlocked tricks

### 4. Between-Turn Actions

Status: Completed on 2026-04-21.

- buy new dice

Implementation details:

- exposed a selected-die workshop in `src/App.tsx` so one die can be inspected and upgraded directly
- added tuned MVP cost helpers for new dice and die ladder upgrades so early runs do not stall immediately
- updated the log feed to record shop, prestige, slot, and reset actions alongside turn resolution events

Exit criteria:

- the player can complete the full loop without editing data manually
- dice growth follows the ladder in `docs/mvp-design.md`

### 5. Persistence

Status: Completed on 2026-04-21.

- save run and meta state to `localStorage`
- auto-save after each state-changing action
- expose reset options for run-only and full-save wipe

Implementation details:

- added `src/game/storage.ts` for versioned `localStorage` load, save, reset, and wipe helpers
- hydrated the app from persisted save data on boot instead of always creating a fresh session
- added auto-save through `useEffect` so every state mutation writes back to the local save slot
- kept run-only reset separate from full-save wipe so permanent progress boundaries stay explicit
- exposed save status and a full-wipe control in the action panel UI

Exit criteria:

- refresh preserves the current run
- resets behave predictably

Implementation details:

- the repository is ready for iterative `v0.0.1x` development

## Suggested Build Order

1. App shell and state scaffolding
2. Resolver and deterministic game logic
3. Between-turn actions
4. Persistence
5. Initial content and unlocks
6. Tests and pacing cleanup

## Definition Of Done For v0.0.1a

- local-first SPA
- one playable run from start to end
- browser refresh persistence
- visible trick and reward breakdowns
- manual reset support
- minimal documentation for running and extending the prototype
